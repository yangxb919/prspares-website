## favicon 优化：881 KB → 26 KB（unused-JS 优化 第二步）

- **时间**: 2026-05-12
- **操作类型**: 性能优化
- **涉及路径**:
  - `public/favicon.png`（覆盖：1024×1024 881 KB → 32×32 1.6 KB）
  - `public/favicon-16x16.png`（新增 973 B）
  - `public/favicon-32x32.png`（新增 1.6 KB）
  - `public/apple-touch-icon.png`（新增 180×180 22 KB）
  - `public/_archive/favicon-1024.png`（归档原图）
  - `src/app/layout.tsx`（[icons metadata](src/app/layout.tsx#L41-L48) 引用 16/32/apple 各自正确尺寸）
  - `src/pages/_document.tsx`（[Pages Router head](src/pages/_document.tsx#L8-L11) 引用同步）
- **变更说明**:
  - 上一步性能复测发现 PSI 报告 `https://www.phonerepairspares.com/favicon.png` Transfer Size 861 KB（实际文件 881 KB），位列首屏第一大资源 — 比 hero webp（143 KB）还大 6 倍
  - 根因：[src/app/layout.tsx:42-46](src/app/layout.tsx#L42-L46) 和 [src/pages/_document.tsx:8-11](src/pages/_document.tsx#L8-L11) 把同一张 1024×1024 PNG 同时声明成 16×16 + 32×32 + shortcut + apple-touch — 浏览器在 head 解析阶段就 fetch
  - 用 sharp 离线从原图（已归档至 `_archive/favicon-1024.png`）生成 16/32/180 三档真实尺寸，分别绑定到正确的 `rel/sizes`
  - `favicon.png` 保留作 32×32 兼容版本（万一外部链接 hard-code 引用）
- **验证**:
  - `npm run build:fast` 通过
- **预期收益**:
  - 首屏移动端节省 ~855 KB（97% 体积削减），LCP 临界资源大幅减少
  - 慢速 4G 模拟下首屏完成时间预计减少 1-2s
  - 桌面端因 favicon 不阻塞 LCP 关键路径，受益较小但仍正向

## nginx 开启 HTTP/2（首屏并发优化 第三步）

- **时间**: 2026-05-12
- **操作类型**: 配置 / 服务器
- **涉及路径**:
  - VPS `/etc/nginx/sites-enabled/prspares-site`（修改 6 行 `listen 443 ssl` → `listen 443 ssl http2`）
  - VPS `/etc/nginx/sites-enabled/prspares-site.bak.20260512`（备份原配置）
- **变更说明**:
  - 上一步性能复测里 PSI 提示 `uses-http2` 可省 mobile 2.1s / desktop 0.7s — nginx 1.18 仍以 HTTP/1.1 处理 TLS 连接，浏览器对同域并发请求受限 6 个
  - 通过 `ssh prspares` 登录 VPS，对 3 个 server block（prspares.xyz / phonerepairspares.com / www.phonerepairspares.com）的 `listen 443 ssl` 加 `http2` 关键字（IPv4 + IPv6 共 6 行）
  - 改前 `sudo cp prspares-site prspares-site.bak.20260512` 备份
  - `sudo sed -i -E 's/(listen (\[::\]:)?443 ssl);/\1 http2;/'` 精确替换
  - `sudo nginx -t` syntax OK（已有 conflicting server name warning 与本次无关，是其他 conf 文件重复声明的遗留）
  - `sudo systemctl reload nginx` 平滑重载（不中断现有连接）
- **验证**:
  - `curl --http2 -I https://www.phonerepairspares.com/` → `HTTP version: 2  code: 200`
  - `curl --http2 -I https://phonerepairspares.com/` → `HTTP/2 301`（apex → www 跳转，HTTP/2 OK）
  - `curl --http2 -I https://prspares.xyz/` → `HTTP/2 403`（bot 拦截，HTTP/2 OK）
  - `systemctl status nginx` active running，1 week 6 days uptime 保持
- **预期收益**:
  - 移动端首屏 ~2.1s 减少（PSI 估算）：多路复用消除同域 6 路并发限制 + header 压缩
  - 桌面端首屏 ~0.7s 减少
  - hero/proof 图、字体、_next 静态 chunks 现在可在单连接复用
- **回滚步骤**（紧急情况）:
  - `ssh prspares` → `sudo cp /etc/nginx/sites-enabled/prspares-site.bak.20260512 /etc/nginx/sites-enabled/prspares-site && sudo nginx -t && sudo systemctl reload nginx`
