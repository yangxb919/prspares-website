# 2026-05-04 操作日志

## 表单反垃圾防护：Turnstile + Honeypot + 速率限制全面铺开

时间：2026-05-04
操作类型：新增 / 修改

### 背景
最近收到大量无效表单提交。项目原本已经接入了 Cloudflare Turnstile（`NEXT_PUBLIC_TURNSTILE_SITE_KEY` / `TURNSTILE_SECRET_KEY` 都已配置），但只在 `QuoteModalEnhanced` 和三个 SEA 落地页（`/wholesale-inquiry`, `/id/wholesale`, `/th/wholesale`）做了客户端展示，**服务端没有强制校验 token**，并且很多表单根本没接 Turnstile。

### 方案
扩展现有 Turnstile，而不是重新装 Google reCAPTCHA。三层防线：
1. **服务端强制 Turnstile token 校验**（RFQ / 落地页询盘必填）
2. **Honeypot 隐藏字段**（一行 `<input>`，机器人填了就拒绝）
3. **IP 速率限制**（复用已有的 `lib/rate-limit.ts`，每 IP 每分钟 5 次）
4. **轻量内容检查**（一次性邮箱黑名单、消息中 URL 数量上限）

新闻订阅（newsletter）在底部 Footer 全站铺开，加 Turnstile 会污染整站 UX，因此**只对 newsletter 加 honeypot + rate-limit + 内容检查，不要求 Turnstile**。

### 新增文件
