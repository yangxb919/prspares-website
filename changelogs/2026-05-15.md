## 23:40 - 修复 wholesale-inquiry 表单 Turnstile 双重验证 bug（begin_form→generate_lead 75% 流失根因）

- **操作类型**：bug 修复
- **涉及路径**：`src/app/wholesale-inquiry/page.tsx`（删除客户端预 verify 代码块 22 行）
- **变更说明**：
  - GA4 数据显示 7 天 begin_form=8 但 generate_lead=2，漏斗损失 75%。诊断后定位到根因：
    - 客户端 [page.tsx:294-310](src/app/wholesale-inquiry/page.tsx) POST `/api/turnstile/verify` 先验一次 → token 被 Cloudflare 消费（一次性 token）
    - 然后 `submitRfqAndNotify` → `/api/send-rfq-email` → [verifyTurnstile.ts:53](src/lib/security/verifyTurnstile.ts) 用同一 token 再调 Cloudflare siteverify → 必然返回 `timeout-or-duplicate` → API 403 → 用户看到 "Captcha verification failed"
  - 修复策略：保留 server-side verify（真正的安全边界），删除客户端预 verify（只提供 UX 反馈、无安全收益）
  - 类型检查 tsc 通过
- **相同 bug 还在 3 个文件，本次未动**（因为有其他人未提交的 isTurnstileEnabled 开关重构）：
  - `src/components/QuoteModalEnhanced.tsx`
  - `src/app/id/wholesale/page.tsx`
  - `src/app/th/wholesale/page.tsx`
  - 等用户合并完那波重构后，需要复查这 3 处是否仍有客户端预 verify，有则一并删除
- **预期效果**：
  - wholesale-inquiry 表单提交成功率从 ~25% 回升到 90%+
  - 每周 lead 数从 ~2 → 估算 6-8
  - GA4 数据 24 小时内可见变化；Ads 转化 24-48 小时反馈给 Smart Bidding
