## 19:00 — 排查 — Google Ads 3 转化但未收到询盘

- 时间：2026-04-07 19:00
- 操作类型：排查/诊断
- 涉及文件：
  - `src/app/api/lp-inquiry/route.ts`（代码审阅）
  - `src/app/api/send-rfq-email/route.ts`（代码审阅）
  - `src/lib/supabase/server.ts`（代码审阅）
  - `public/lp/google-ads-factory-direct.html`（代码审阅）
  - `supabase/migrations/202602230001_create_rfqs_table.sql`（代码审阅）
- 排查结论：
  - 根因 1：`/api/lp-inquiry` 对 Supabase 写入和邮件发送都使用 try/catch 仅记录日志，无论后端两步是否成功都返回 `{success:true}`（commit 9dbbd4b 引入），导致前端"假成功"。
  - 根因 2：着陆页表单 JS 在 `response.ok` 分支即触发 GA4 `generate_lead` + Google Ads `ads_conversion`，与邮件送达解耦——这就是 Google Ads 计 3 转化但用户收件箱为空的原因。
  - 嫌疑链路：① SMTP_TEST_TO 未设置时回退到 SMTP_USER 导致"自己发自己"被反垃圾过滤；② VPS 上 SMTP 调用静默失败；③ Supabase 写入因 env/RLS 失败。
- 操作：
  - 向生产 API 提交了一条诊断请求 `name=DIAG TEST claude`/`source=diagnostic-claude`，API 返回 200 成功，可用于在 Supabase 和 SMTP 邮箱核对落地情况。
- 后续验证（已通过 ssh prspares + Supabase REST API 完成）：
  - VPS pm2 日志反复出现 `[LP Inquiry] Email notification failed: {}` —— 内部 fetch 被 Next.js 中间件 403 拦截
  - 生产 Supabase 不存在 `public.rfqs` 表（42P01），迁移从未应用；存在 `contact_submissions` 但为空
  - 用户邮箱（含垃圾箱）无任何 RFQ 邮件
- 最终根因（三 bug 叠加）：
  1) 生产 Supabase 没有 rfqs 表 → 写入失败被吞
  2) lp-inquiry 内部 fetch localhost send-rfq-email → 被中间件 403 → 邮件失败被吞
  3) 两步失败都 try/catch 仅 log，API 仍返回 success → 前端触发 Google Ads 假转化
- 影响：3 个 Google Ads 转化对应的潜在客户信息完全丢失，无可挽回。
- 修复方案待用户确认后实施：抽 sendMail 到 lib 直接调用、改写 contact_submissions、失败时返回 5xx。
