## 00:00 — 修复 / 配置

### 操作类型：修复 + 配置

### 涉及文件
- `src/middleware.ts`（修改）

### 变更说明

**问题诊断：** 分析 GA4 Weekly/Daily 数据，发现两大 P0 问题：
1. 从 3/23 起每天 39-58 个 Direct bot sessions（Singapore datacenter，0s engagement），导致数据严重失真
2. GA4 转化事件（form_submit/contact_click/chat_start）计数与 page_view 完全相同，属于 Enhanced Measurement 自动触发

**方案 A — Middleware Bot 过滤（代码层面）：**
- 在 `src/middleware.ts` 中添加 bot 检测逻辑（UA pattern 匹配 + IP 前缀屏蔽）
- 允许 Googlebot、Bingbot 等 SEO 友好爬虫通过
- 拦截 headless chrome、scraper、已知恶意 bot（返回 403）
- 更新 matcher 配置为全路由匹配（排除静态资源）

**方案 C — GA4 Data Filter（后台配置）：**
- GA4 Admin → Data Filters → Internal Traffic filter 状态从 Testing 改为 **Active**
- 后续识别到 bot IP 后可通过 Google Tag → Define Internal Traffic 规则进一步排除

**方案 E — GA4 Enhanced Measurement 修复（后台配置）：**
- GA4 Admin → Data Streams → Enhanced Measurement → **关闭 Form Interactions**
- 此前 Form Interactions 开启导致 GA4 在每次页面加载时自动触发 form_submit 事件，与代码中手动事件冲突
- 关闭后 Enhanced Measurement 从 "+4 more" 变为 "+3 more"
