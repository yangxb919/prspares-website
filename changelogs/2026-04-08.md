## 数据 — 回补 04-07 Daily（04-04~06 backfill 受阻）

- 操作类型：新增 / 数据回补
- 涉及文件：
  - `Analytics/daily/2026-04-07.md`（新增）
- 处理：
  1. 通过 GA4 Reports snapshot "Yesterday" 预设拉取 2026-04-07 单日数据，写入 daily 文件：active_users 234 / new_users 230 / event_count ≈2700 / avg_engagement 4s / sessions_total 235 / 7 条 top pages / 5 条 sessions_by_source / 7 个 city。Singapore 占比 86.3% 仍异常。
- backfill 受阻说明（04-04 / 04-05 / 04-06）：
  - GA4 日期选择器使用 Material CDK Virtual Scroll Viewport 渲染年份月份列表，仅渲染可视区域（Jan 2015 起 ~12 个月），需要滚动 ~130 个月才能定位 April 2026。CDP 下尝试均失败：`element.scrollTop` 设置无效、`scrollTo` 无效、`dispatchEvent('wheel')` 无效、`scrollIntoView` 无效、Angular component 实例 (`window.ng.getComponent`) 在生产构建下不可用。
  - URL hash 自定义日期参数尝试 `_u..date00=YYYYMMDD` / `_u.dateOption=custom&_u.date00=...` / `_u..dateRange=YYYYMMDD,YYYYMMDD` / `_r.dateRange00=...` 等多种组合均被 GA4 忽略或抛 "Unknown error loading report" 弹回 Home。
  - mat-datepicker 的文本输入框尝试通过 React-style value setter + input/change/blur 事件注入日期，input.value 显示更新但 Material 的 dateAdapter 未触发解析，Apply 后日期未变。
  - 结论：当前 CDP + Reports snapshot 路径无法稳定切换到任意单日；只有 today / yesterday 两个预设可用。建议后续改用 GA4 Data API（需 OAuth）或人工在浏览器中手动选择日期后让 Claude 拉取。
- 04-08 写入后连续性更新：04-04 / 04-05 / 04-06 仍缺失，03-31 历史缺失；04-07 / 04-08 已补齐。
