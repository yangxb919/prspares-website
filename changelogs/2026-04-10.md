## 修复 — Bot 流量过滤（第 3 轮，根因修复）

- 操作类型：修复
- 背景：前两轮修复（03-26 middleware UA/IP 屏蔽 + 03-29/04-01 analytics.ts engagement gate）均未有效过滤 Singapore bot 流量（日占比 77–92%）。经 ai-pair 多模型审查（Codex GPT-5.4 + Gemini 2.5-flash）确认根因：
  1. GTM 的 GA4 Configuration Tag 自动发 page_view，完全绕过 analytics.ts 的 engagement gate
  2. analytics.ts 自身的 `fireDelayedPageView()` 也绕过了自己的 `passesEngagementGate()` 函数
- 处理：
  1. **`src/lib/analytics.ts`**（修改）：
     - page_view gate 从 `{ minTimeOnPage: 3 }` 改为 `{ requireHuman: true, minTimeOnPage: 3 }`
     - `fireDelayedPageView()` 增加 `passesEngagementGate('page_view')` 检查，不通过则 2 秒后重试，不再无条件发送
  2. **GTM（GTM-TTBMN854）**（配置修改）：
     - 在 "Google Analytics 4" tag（Google 代码，ID: G-BPQXLYL76G）的配置参数中添加 `send_page_view = false`
     - 发布为版本 12（v12 - Disable auto page_view for bot filtering），2026/04/10 16:48 生效
- 效果预期：
  - Bot 没有真实鼠标/键盘/触摸/滚动 → 不触发 markHuman() → page_view 不发送 → GA4 不记录
  - GTM 不再自动发 page_view → 堵住 GTM 绕过通道
  - 真实用户正常浏览（移动鼠标 3 次 / 滚动 >5% / 键盘输入 / 触摸）→ 3 秒后正常记录 page_view
- 后续观察：2-3 天后对比 GA4 Singapore 流量是否显著下降
