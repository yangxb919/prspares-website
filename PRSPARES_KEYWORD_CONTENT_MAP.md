# PRSPARES 关键词地图 + 页面分工 + 内容集群表

**用途：** 给 Claude Code 和后续内容执行使用  
**目标：** 让页面、关键词、内容、CTA 形成统一结构，而不是分散优化

---

# 一、关键词策略总原则

PRSPARES 的关键词策略不应再采用“泛发内容”的方式，而应采用以下分工：

- **首页 / 产品页 / 分类页 / 专题页**：承接商业采购意图词
- **博客 / 指南内容**：承接信息搜索词与决策比较词
- **所有内容页**：最终回流到产品页与询盘页

# 八、文章页统一转化规则

每篇文章必须包含：

## 1. 中段 CTA
根据主题插入：
- Need wholesale phone screens?
- Looking for a reliable battery supplier?
- Get a wholesale quote from PRSPARES

## 2. 文末 CTA
统一导流：
- Browse Products
- Get Wholesale Quote

## 3. 内链要求
每篇至少链接到：
- 1 个对应分类页
- 1 个 inquiry 页面
- 2 篇相关内容页

## 4. 商业语境要求
即使是 informational 内容，也必须尽量贴近：
- repair shop
- buyer
- distributor
- sourcing decision

避免纯 C 端教程语气。

# 九、建议弱化的关键词方向

以下方向除非有明确商业承接，否则不应成为主力：

- 纯消费者维修教程词
- 纯娱乐性行业资讯
- 与批发采购无关的泛流量词
- 无法回流到产品与询盘页的低商业价值内容

原则：
**宁可少做，也不要做一堆不能转化的流量。**

# 十一、Claude Code 执行要求

Claude Code 在修改代码时，应按照以下方式使用本文件：

1. 先确认每个核心页面服务哪类关键词
2. 再确认页面 CTA 与内容职责
3. 不要让多个页面抢同一类词却没有分工
4. 所有 blog 内容都必须有商业回流路径
5. 新增或重写页面时，优先保障：
   - 页面角色明确
   - 关键词明确
   - CTA 明确
   - 与其它页面关系明确
